# Таблица ключевых понятий Day06

| No | Понятие | Объяснение | Пример | Практическое значение |
|---|---|---|---|---|
| 1 | Anaconda | Установщик Red Hat Enterprise Linux, который управляет ручной установкой системы. | Экран Installation Summary | Через него выбираются диск, сеть, пользователи и профиль установки. |
| 2 | binary ISO | Полный установочный ISO-образ с пакетами внутри. | `rhel-10.0-x86_64-dvd.iso` | Подходит для установки без постоянного доступа к сети. |
| 3 | boot ISO | Минимальный ISO-образ для загрузки установщика с получением пакетов из сети. | `rhel-10.0-x86_64-boot.iso` | Удобен, когда пакеты берутся с сетевого репозитория. |
| 4 | Minimal Install | Минимальный набор пакетов RHEL без лишней графической среды. | Выбор профиля в Anaconda | Делает сервер легче и уменьшает поверхность настройки. |
| 5 | tmux | Терминальный мультиплексор, используемый установщиком для отдельных окон логов. | `tmux list-windows` | Помогает смотреть anaconda.log, storage.log и packaging.log во время установки. |
| 6 | Kickstart | Текстовый файл автоматической установки RHEL. | `kickstart.cfg` | Заменяет ручные действия в Anaconda заранее заданными директивами. |
| 7 | ksvalidator | Инструмент проверки синтаксиса Kickstart. | `ksvalidator kickstart.cfg` | Позволяет найти ошибки до запуска установки. |
| 8 | inst.ks= | Параметр загрузки, указывающий URL или путь к Kickstart-файлу. | `inst.ks=http://servera.lab.example.com/kickstart.cfg` | Запускает автоматическую установку по подготовленному сценарию. |
| 9 | autopart | Директива автоматической разметки диска в Kickstart. | `autopart --type=lvm` | Быстро создает типовую LVM-разметку. |
| 10 | %packages | Секция Kickstart со списком пакетов и групп пакетов. | `%packages`, `@Core`, `vim` | Гарантирует, что после установки будут нужные утилиты. |
| 11 | %post | Секция Kickstart с командами после установки. | `%post`, `mandb`, `systemctl enable sshd` | Используется для финальной настройки установленной системы. |
| 12 | Podman | Утилита управления контейнерами и образами без Docker daemon. | `podman run`, `podman ps` | Позволяет запускать контейнеры от обычного пользователя. |
| 13 | container image | Неизменяемая основа контейнера с файловой системой и командой запуска. | `registry.lab.example.com:5000/ubi10/ubi` | На основе образа создаются контейнеры. |
| 14 | container instance | Запущенный или завершенный экземпляр, созданный из образа. | `podman ps -a` | Показывает фактическое состояние конкретного запуска. |
| 15 | Containerfile | Файл инструкций для сборки контейнерного образа. | `FROM ...`, `CMD ...` | Делает сборку `my_image:1.0` и `my_image:1.1` воспроизводимой. |
| 16 | detached container | Контейнер, запущенный в фоне без удержания терминала. | `podman run -d --name my_webserver ...` | Нужен для сервисов, например веб-сервера. |
| 17 | registry | Хранилище контейнерных образов и тегов. | `registry.lab.example.com:5000` | Через registry выполняются `podman login`, pull и push образов. |
| 18 | image tag | Метка версии контейнерного образа. | `my_image:1.0`, `my_image:1.1` | Позволяет хранить несколько версий одного образа. |
